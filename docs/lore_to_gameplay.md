# SOARA V2 - Traduction Lore -> Gameplay

Version: 2.0
Statut: Design reference

## 1. Principes de design
- P1: La decision doit produire la consequence.
- P2: L'energie est une contrainte centrale.
- P3: Le tempo est une arme.
- P4: Les factions et races doivent se sentir differentes en combat.

## 2. Races -> Mecanique
### Humain
- Identite: polyvalence et progression technique.
- Mecanique:
  - meilleure flexibilite de build techniques
  - bonus d'apprentissage/maitrise de sequences
  - courbe d'Ascension (mid/late progression)

### Gobelin
- Identite: pression courte, coordination, adaptation.
- Mecanique:
  - +3 d'initatiative en plus du jet(ne peux pas dÈpasser le max du timer), PV reduit, Atk Resuit def reduit, esq forte)
  - effets de drain/pression energie
  - pics d'efficacite en enchainements courts

### Orc
- Identite: endurance, impact frontal, perseverance.
- Mecanique:
  - DEF et PV et ATK de base leve, ESQ reduite
  - penalite reduite sur engagements longs
  - forte valeur des sequences directes

## 3. Aura -> Mecanique
- Aura = meta-maitrise, pas magie.
- Impacts concrets:
  - reduction de cout conditionnelle
  - precision de timing (fenetres plus fiables)
  - stabilisation des sequences complexes

## 4. Combat -> Mecanique
- Technique = sequence de symboles.
- Chaque tour:
  - intention (technique en attente)
  - validation (timer / Tempo)
  - resolution deterministe via table et fiches entites
  - trace dans le log
- Log = donnee de lecture/relecture, pas decoration UI.

## 7. Definition de fini (combat Soara)
- Le joueur peut expliquer pourquoi il a gagne/perdu.
- Le log permet de retracer chaque consequence.
- La victoire n'apparait pas comme un coup de chance.
